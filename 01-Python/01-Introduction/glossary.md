# Glossary

| Term | Definition |
| --- | --- |
| Assignment | Binding a name to a value, usually with `=`. |
| Boolean | A logical value: `True` or `False`. |
| Built-in function | A function provided by Python, such as `len` or `sum`. |
| Comment | Text beginning with `#` that Python ignores during execution. |
| Dynamic typing | Runtime values have types; names are not permanently fixed to one type. |
| Expression | Code that produces a value, such as `price * quantity`. |
| Function | A reusable block of code that performs an operation. |
| Indentation | Leading whitespace used to define Python code blocks. |
| Interpreter | The program that executes Python code. |
| Library | Reusable code distributed for a specific set of capabilities. |
| List | An ordered, mutable collection written with square brackets. |
| Notebook | An interactive document containing executable code cells and rich text. |
| Object | A value in Python with an identity, type, and data. |
| Script | A Python source file, commonly ending in `.py`. |
| Syntax | The rules that determine how valid Python code is written. |
| Variable | A name that refers to an object. |
