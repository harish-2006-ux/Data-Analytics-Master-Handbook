# Chapter Board Manifest

This file mirrors the GitHub Project board for the handbook. The board columns are fixed as:

`Backlog` | `Ready` | `Writing` | `Review` | `Completed`

Each row becomes one GitHub Project card. The board columns remain `Backlog`, `Ready`, `Writing`, `Review`, and `Completed`. The chapter lifecycle status is tracked separately so editorial progress is visible inside a board card.

## Chapter Lifecycle Status

| Lifecycle status | Meaning |
| --- | --- |
| Draft | Initial manuscript is being created. |
| Review | Technical and learner-focused review is in progress. |
| Revision | Review corrections are being applied. |
| QA | Final code, formatting, links, and reference checks are in progress. |
| Published | DAMS v2.0 is satisfied and the chapter is approved as v1.0. |

`Completed` requires `Published` lifecycle status and 5-star Book Quality.
